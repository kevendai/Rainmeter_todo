# 待办 / 已办磁贴

## 交互

- `+`：打开无终端的深色编辑卡片，可输入标题、跳转目标、开始时间、截止时间和备注。
- `☰`：打开全部任务管理器，按“逾期 → 未开始 → 待办 → 已办”分组排序，同一类别内按添加时间从新到旧排列；可完成/恢复、修改或删除，勾选任务后还可批量完成、批量恢复或批量删除。
- `○`：标记完成。
- `↶`：恢复为待办。
- `✎`：修改任务内容和时间限制。
- `×`：确认后删除。
- `↻`：手动检查 daily_arxiv 前五篇。
- 点击任务文字：打开网页、文件、文件夹或应用。

新增和修改待办时可以填写备注，并通过圆角标签按钮增删通用标签。备注和标签只保存在任务数据中，不显示在桌面磁贴及悬浮提示中。

任务 JSON 使用统一的 `labels` 数组，不增加来源专用状态字段。程序导入论文时添加 `论文`，从日程转换待办时添加 `日程`；手动完成论文时添加 `已读`；06:00 日切自动完成时添加 `自动归档`。恢复论文会移除 `已读` 和 `自动归档`，重新等待阅读。因此后续回顾时可用“包含 `论文` 且不包含 `已读`”准确筛出未阅读论文；修改页面也允许手动修正这些标签。

## 时间限制

开始时间和截止时间均可留空：两者都为空时，任务创建后一直显示；只有截止时间时，任务立即显示并在超时后标红；只有开始时间时，任务到时才出现，此后一直保留；两者都有时，任务到开始时间出现，超过截止时间后标红。截止时间不能早于开始时间。

待办首先按截止时间从早到晚排序，有截止时间的排在无截止时间之前；截止时间相同时，再按创建时间从早到晚排序。无截止时间的任务也按创建时间从早到晚排序。尚未到开始时间的任务不会出现在桌面磁贴中，但可通过顶部 `☰` 管理按钮查看、修改或删除。皮肤每分钟检查一次时间边界，未完成的逾期任务会显示红色标题、圆圈和逾期提示。旧任务没有时间字段时继续按无时间限制任务处理。

编辑器中的日期和时间采用统一的深色输入框，日期格式为 `YYYY-MM-DD`，时间为 24 小时制 `HH:mm`；勾选时间限制时会自动填入当天日期。

已办任务按实际完成时间保留 30 天，超过 30 天后会在下一次加载或操作磁贴时自动清理。缺少有效完成时间的旧任务不会被自动删除。

已办区域以每天早上 06:00 为换日时间，只显示当前完成周期内的任务：06:00 之前显示前一天 06:00 至当前完成的任务，06:00 之后显示当天 06:00 至当前完成的任务。更早的已办仍按 30 天规则保存在数据中，只是不显示在磁贴上。

每天 06:00 日切时，前一完成周期内仍处于待办状态的 arXiv 论文会自动改为已办，并标记 `自动归档` 而不是 `已读`。自动完成时间记录在新周期分界之前，因此这些论文不会出现在新一天的已办区域，但仍以 `completed: true` 保留在任务 JSON 中，直至触发 30 天清理。Rainmeter 持续运行时由包含日期的分钟变化事件每分钟触发一次轻量 Render，因此跨过 06:00、电脑睡眠后唤醒或错过整点时都会在下一次检查自动切换已办窗口。后端只在 06:00 分界已经到达且存在昨日待办论文时改写数据；如果当时没有运行，也会在下次启动或操作时补做日切。

手动恢复已办论文表示重新激活阅读任务。恢复时会将该论文的 `created_at` 更新为恢复时间，使它在刷新后继续保持待办，并延续到下一个 06:00 日切；若恢复后手动完成，`completed_at` 会记录当前时间，因此它会稳定显示在当前完成周期的已办区域。

## arXiv 自动逻辑

Rainmeter 启动或皮肤加载时，仅在本地时间 08:00–20:00 检查。如果待办中没有论文，则读取 `daily_arxiv` 当日评分 JSON，按摘要分数降序、标题分数降序取前五篇。当日不会重复自动添加，arXiv ID 也会全局去重。

论文导入时直接把任务标题保存为 `(摘要分数) 翻译后标题`，备注保存论文英文原标题和 arXiv ID，点击后打开对应的 arXiv HTML 页面。任务数据不再保存 `abstract_score`、`translated_title` 或 `arxiv_id` 专用字段。如果本地没有当日 JSON，后端会按原 `daily_arxiv_tools.py` 的 File Browser 配置尝试下载，不会把账号或密码复制到本项目。

Rainmeter 自行下载的论文 JSON 放在 `Todo/@Resources/PaperCache/`，仅保留 14 天。清理只匹配 `*_papers.json`，不会删除 `daily_arxiv` 项目自己的历史数据。

## 论文标题翻译

导入 arXiv 论文时调用腾讯云 TMT `TextTranslate` 将英文标题翻译为中文。磁贴显示 `(摘要分数) 中文标题`，英文原标题和 arXiv ID 保存在备注中。单篇翻译失败时标题回退为英文，不会阻断导入。

腾讯云凭据使用 Windows DPAPI CurrentUser 加密，密文只存放在实际 Rainmeter 运行目录的 `Todo/@Resources/translation.secret`，不放入源码、任务 JSON 或日志。

更换凭据时运行 `powershell -ExecutionPolicy Bypass -File scripts/Set-TranslationSecret.ps1`，SecretKey 会以隐藏输入方式读取并重新加密。

## 数据和部署

- 源码：`skins/Todo/`
- 待办数据：`skins/Todo/@Resources/tasks.json`
- 部署：`powershell -ExecutionPolicy Bypass -File scripts/Deploy-Todo.ps1 -Activate`

`TodoHost.exe` 由部署脚本从 `backend/Common.cs` 与 `backend/TodoApp.cs` 构建。任务窗口、JSON 原子写入、DPAPI 解密、arXiv 下载和腾讯翻译都直接在 C# 后端中执行；Rainmeter 事件不再启动 PowerShell。PowerShell 仅保留部署、图标生成与凭据设置用途。
