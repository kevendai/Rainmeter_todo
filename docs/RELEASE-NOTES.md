# Release Notes

## 1.3.4 - 2026-06-29

- 统一版本来源：根目录 `VERSION` 现在驱动 Rainmeter 皮肤管理器元数据、后端 About/更新检查版本、发布包 `manifest.json` 和包文件名，减少发版时版本号漂移。
- 发布包新增 `full` / `lite` 两个 `.rmskin` 初始安装包，已安装 Rainmeter 的用户可以直接用 `SkinInstaller.exe` 安装；自动更新仍使用原有 zip + updater 路径以保留旧数据。
- 修复本机部署脚本漏编译拆分后的 `Todo*.cs` / `Calendar*.cs` 文件风险，并在部署时同步写入 `app-version.txt` 和皮肤 `Version=`。

## 1.3.3 - 2026-06-29

- 优化设置页“检查更新”流程：设置页已经确认更新后，独立升级器会直接下载并部署，不再额外弹出第二个确认框。
- 升级器下载更新包时改用兼容的 BasicParsing 网络请求路径，避免 Windows PowerShell 走旧网页解析组件时出现额外安全/脚本提示。
- 保持旧升级器兼容：`Install-Skins.ps1` 根入口、包内 `Updater/RainmeterDesktopWidgetsUpdater.ps1`、raw tag 包路径和 GitHub Release 资产布局不变。

## 1.3.2 - 2026-06-29

- 新增轻量服务层拆分：`TodoService.cs` 承接待办交互式命令和锁定状态执行器，`CalendarService.cs` 承接日历同步、CalDAV 保存/删除和日程转待办相关的编排逻辑。
- 保持 `TodoApp.cs` / `CalendarApp.cs` 更接近命令入口，协议解析、ICS 读写、窗口绘制和 Rainmeter 渲染继续由各自模块负责，为后续脱离 Rainmeter 宿主预留边界。
- 继续保留 1.2.x/1.3.x 升级器兼容：发布包根目录仍提供 `Install-Skins.ps1`，包内仍包含 `Updater/RainmeterDesktopWidgetsUpdater.ps1`，raw tag 包路径和 `manifest.json` 结构不变。

## 1.3.1 - 2026-06-29

- 拆分 Calendar 后端结构：`CalendarApp.cs` 保留命令入口，日历状态存储、CalDAV 同步、日程转待办规则、ICS 读写、WinForms 管理窗口和 Rainmeter 渲染分别移动到独立的 `Calendar*.cs` 模块。
- 打包和后端测试脚本改为自动编译所有 `Calendar*.cs`，与 1.3.0 的 Todo 模块化方式一致，降低后续继续拆分时漏编译新文件的风险。
- 保持现有日程 UI、CalDAV 数据格式、Rainmeter 皮肤入口和旧升级器包路径兼容；本版本只调整代码组织，不迁移用户数据。

## 1.3.0 - 2026-06-29

- 拆分 Todo 后端结构：`TodoApp.cs` 现在保留命令入口，待办数据存储、业务规则、Rainmeter 渲染、设置/管理窗口和更新/论文同步逻辑分别移动到独立 `Todo*.cs` 模块，为后续从 Rainmeter 宿主中脱离做准备。
- 保持现有 WinForms 界面、Rainmeter 磁贴、数据文件和命令参数兼容；本版本不主动改版 UI，也不迁移用户数据格式。
- 打包和测试脚本改为编译所有 `Todo*.cs` 源文件，避免后续继续模块化时遗漏新文件。

## 1.2.4 - 2026-06-29

- 调整“检查更新”交互：按钮先在设置窗口内轻量检查 GitHub 最新 tag，只有检测到新版本时才弹出更新确认并启动独立升级器；已是最新版本时只更新设置页状态，不再额外打开 PowerShell 窗口或弹出“Already on latest version”对话框。

## 1.2.3 - 2026-06-29

- 修复 1.2.2 更新器在安装包入口和升级器自升级后部署本体时传递 `-Activate` 开关参数的方式，避免 Windows PowerShell 将 `System.String` 转换为 `SwitchParameter` 失败。

## 1.2.2 - 2026-06-29

- 新增独立升级器脚本 `Updater/RainmeterDesktopWidgetsUpdater.ps1`。`TodoHost.exe` 的“检查更新”按钮现在只负责启动升级器，版本检查、下载、解压、部署都由升级器脚本处理，后续升级策略可以不重编主程序。
- 发布包继续保留 `Install-Skins.ps1` 兼容入口，但安装入口会委托给包内升级器执行部署；从旧版本更新到 1.2.2 时会同时把独立升级器安装到 `Todo/@Resources/Updater/`。
- 升级器支持先从新包中更新已安装升级器，再由已安装升级器部署本体，为之后 UI 与后端分离后的升级流程预留兼容路径。

## 1.2.1 - 2026-06-29

- 新建日程不再默认启用“15 分钟前”提醒；只有用户手动选择常用提醒或添加自定义提醒时，才会写入本地日程和 CalDAV `VALARM`。
- CalDAV 日程保存成功后会立即刷新本地日程缓存；周期日程修改不再先清空同系列缓存，避免保存后短暂消失、必须手动同步才重新出现。
- 保存 CalDAV 凭据时会清空旧日历发现缓存；同步时也只复用属于当前保存服务器的 `calendar_url`，避免地址改成新服务器后又沿用旧服务器。

## 1.2.0 - 2026-06-28

- CalDAV 设置不再内置默认服务器地址；保存、测试连接和后台发现都要求用户显式填写 CalDAV 地址，避免误连到开发环境服务器。
- 新建/编辑日程的提醒区改为常用提醒胶囊 + 展开区：常用提醒直接勾选，自定义“开始前提醒”通过添加提醒弹窗创建，CalDAV 额外格式提醒只在展开区显示并允许删除。
- 删除 CalDAV 额外提醒后保存会二次确认，确认后会从远端 VALARM 中移除；这些结束前提醒或固定时间提醒仍会被解析和保留，但界面不支持重新创建或修改。
- 日程管理的“今天 / 未来7天 / 全部时间”分段按钮修复鼠标经过后背景停留为白色的问题，hover 后会恢复到正确的选中或灰色状态。
- 新建/编辑日程新增提醒设置，支持 5 分钟前、15 分钟前、30 分钟前、1 小时前、5 小时前、1 天前等多个开始前提醒，并写入 CalDAV `VALARM`。
- CalDAV 提醒解析支持多个 `VALARM`，包括 `TRIGGER;VALUE=DATE-TIME` 这类绝对时间提醒；非“日程开始前”的提醒在编辑界面仅允许删除，不允许新增或修改，保存时未删除会原样保留。
- 日程转待办继续使用日程开始时间和最早提醒时间中更早的一个作为待办开始时间；自动转待办扫描也会纳入已到提醒日期的未来日程，避免提前提醒的周期日程等到发生当天才生成待办。

## 1.1.7 - 2026-06-28

- 修复标准 Rainmeter 安装无法通过内置更新定位目录的问题：更新器现在按当前皮肤所在的 `Skins` 目录定位皮肤库，不再要求皮肤库上级目录必须包含 `Rainmeter.exe`。
- 安装脚本支持便携安装目录、`Documents\Rainmeter` 皮肤库目录或直接输入 `Skins` 目录；`Rainmeter.exe` 会从便携目录、正在运行的 Rainmeter 进程或注册表中单独查找，用于重启和刷新皮肤。

## 1.1.6 - 2026-06-28

- 修复日程管理面板点击“刷新同步”后按钮一直停留在后台同步状态的问题；同步完成后会自动重新读取缓存并恢复按钮。
- 修复缓存日历地址与设置页 CalDAV 地址不一致时，事件资源 href 可能回退到旧服务地址的问题，事件地址现在跟随实际查询的日历 URL 解析。
- 更新器解压安装包后会对本次解压出的文件执行解除阻止，安装脚本复制到 Rainmeter 目录后也会清理文件的 Zone.Identifier，避免其他电脑上打开 `TodoHost.exe` / `CalendarHost.exe` 时出现“未知发布者”安全警告。
- 自动更新完成后只清理本次创建的临时解压目录，以及文件名匹配 `rainmeter-desktop-widgets-*.zip` 的本次下载包，不会扫描或删除其它临时文件。
- 安装脚本复制新文件前会先关闭当前 Rainmeter 目录下的 `TodoHost.exe` 和 `CalendarHost.exe`，避免日程管理、编辑或后台同步窗口占用旧程序导致只更新版本号、日程端未替换的半更新状态。

## 1.1.5 - 2026-06-27

- 优化日程管理窗口启动路径：管理和设置界面打开时只短暂等待同步锁，后台正在同步时也会先显示本地缓存，避免 CalDAV 网络请求拖慢窗口出现。
- 日程管理里的“刷新同步”改为后台启动同步进程，不再在窗口 UI 线程里直接执行网络同步，点击后窗口保持可操作。
- CalDAV 同步优先复用已缓存的日历地址，减少每次同步前重复发现日历的开销；缓存地址失效时再回退到服务发现。
- CalDAV 未来 21 天拉取从逐日 21 次查询改为一次 21 天区间查询，保留周期日程展开能力，同时显著减少网络往返。
- 管理窗口关闭时只有发生新建、编辑或设置变更才触发重新渲染和刷新，减少无改动关闭后的拖尾卡顿。

- 改进待办编辑器的标签选择：新增/修改待办时可直接维护 `论文`、`日程`、`已读`、`自动归档` 等通用标签；标签区默认保持单行紧凑，展开时浮在备注区域上层，收起后继续编辑备注。

- 完善今日日程到待办的一次性/系列转换：CalDAV 提醒映射为待办开始时间，日程结束映射为截止时间，转换备注会保留原始时间、地点和说明，会议链接优先作为可点击目标。
- 修复 CalDAV UTC 时间解析导致的 8 小时时差问题，并在更新遇到 stale ETag 时重新读取当前资源后再重试。
- 提升桌面磁贴渲染清晰度：部署后保持不透明窗口并启用 Rainmeter DPI 感知，减少高 DPI 桌面上的模糊显示。
